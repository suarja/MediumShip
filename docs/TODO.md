Media management
Aujourd’hui, les contenus reposent encore sur des URLs collées à la main. Le prochain vrai gain produit est :

upload de cover image
validation des URLs audio/video
champs mieux typés selon article / episode / video
suppression des entrées fragiles type Google Images ou liens non directs
Workflow éditorial plus net
Le socle draft / published / archived existe, mais il manque encore le confort opérable :

filtres par statut
dates de publication éditables
duplication de contenu
confirmations d’archivage
meilleure liste/table de contenus côté CMS
Preview plus crédible
La preview est encore simple. Le bon prochain slice serait :

preview feed
preview détail mobile par type
preview tenant settings en direct
badge clair draft vs published
Authz admin un peu moins minimale
Si plusieurs personnes doivent toucher au CMS :

rôle admin explicite côté Convex
bootstrap du premier admin gardé
blocage propre des non-admins partout
Je ne lancerais pas encore :

premium
downloads offline
member sync
analytics lourde
Mon choix recommandé : faire maintenant le slice media management, parce que c’est le point le plus faible du système actuel et celui qui améliore immédiatement fiabilité CMS + rendu mobile.

Si tu veux, je peux enchaîner directement sur :

upload d’images de cover
formulaire contenu mieux typé par format
validation stricte des sources audio/video
preview média propre dans le CMS
